# 页面跳转开发规范

本文档约定：**源模块**（调用方）如何跳转到**目标模块**的指定页面。不使用前端 SPA 路由，采用**整页跳转**：`window.open(URL, '_blank')`，参数通过 **Query 参数** 传入。

---

## 一、适用场景

- 从模块 A 的页面跳转到模块 B 的某个具体页面（如：从 01288 跳转到 01206 的 orderEntryNo 页面）。
- 目标模块以 **path 段（routeName）** 暴露可跳转的页面，源模块按约定 URL 拼装后直接打开新窗口。

---

## 二、URL 约定

### 2.1 固定格式

```
{contextPath}/websms{moduleNo}/maintenance/{routeName}?{queryParam_1}={value_1}&{queryParam_2}={value_2}&....
```

| 占位符 | 说明 |
|--------|------|
| **contextPath** | 应用上下文路径，由页面通过 `request.getContextPath()` 或前端等效方式获取 |
| **moduleNo** | 目标模块编号，如 `01206`、`01288`、`012C8`、`01281` |
| **routeName** | 目标页面路由名，由目标模块在文档中公布（如 `orderEntryNo`、`list`） |
| **queryParam_* / value_*** | 需要带给目标页面的参数，以 Query 方式传入 |

### 2.2 示例：跳转到 01206 的 orderEntryNo

**URL：**

```
{contextPath}/websms01206/maintenance/orderEntryNo?corporate_code=X&table_type=order
```

**前端写法：**

```javascript
var url = contextPath + '/websms01206/maintenance/orderEntryNo?corporate_code=X&table_type=order';
window.open(url, '_blank');
```

---

## 三、源模块实现要点

1. **不使用 SPA 路由**  
   不通过前端路由（如 Vue Router、React Router）做模块间跳转，而是直接使用上述 URL 进行**整页打开**。

2. **使用 window.open(..., '_blank')**  
   统一在新标签页打开，避免覆盖当前页。

3. **参数以 Query 方式传入**  
   所有需要传给目标页面的参数，拼在 URL 的 `?` 后面，格式：`key1=value1&key2=value2&...`。  
   参数值需使用 `encodeURIComponent` 编码，避免特殊字符导致错误。

4. **拼 URL 即可，无需调用 routeQuery 等接口**  
   源模块只需按约定拼出完整 URL，然后 `window.open(url, '_blank')`。

---

## 四、前端示例

### 4.1 无参数

```javascript
var url = contextPath + '/websms01206/maintenance/orderEntryNo';
window.open(url, '_blank');
```

### 4.2 带 Query 参数

```javascript
var base = contextPath + '/websms01206/maintenance/orderEntryNo';
var params = 'corporate_code=X&table_type=order&entryNo=' + encodeURIComponent(entryNo);
var url = base + '?' + params;
window.open(url, '_blank');
```

### 4.3 参数较多时（拼 Query 字符串）

```javascript
var params = {
  corporate_code: 'X',
  table_type: 'order',
  entryNo: 'ENT001'
};
var qs = Object.keys(params)
  .filter(function(k) { return params[k] !== undefined && params[k] !== null; })
  .map(function(k) { return encodeURIComponent(k) + '=' + encodeURIComponent(params[k]); })
  .join('&');
var url = contextPath + '/websms01206/maintenance/orderEntryNo?' + qs;
window.open(url, '_blank');
```

### 4.4 一行写法

```javascript
window.open(contextPath + '/websms01206/maintenance/orderEntryNo?corporate_code=X&table_type=order', '_blank');
```

## 五、目标模块实现要点（提供可跳转页面的一方）

1. **为每个对外暴露的页面提供 GET 映射**  
   - 在对应 Controller 下（如 `@RequestMapping("/websms01206/maintenance")`）增加 path 与 **routeName** 一致的 GET 方法。  
   - 例如：routeName 为 `orderEntryNo`，则提供 `@GetMapping("orderEntryNo")`（或 `@RequestMapping(value = "orderEntryNo", method = RequestMethod.GET)`），方法内接收需要的 Query 参数，渲染对应 JSP/视图。

2. **Query 参数**  
   - 通过 `@RequestParam` 接收调用方传入的 `corporate_code`、`table_type`、`entryNo` 等参数，按业务使用即可。

3. **routeName 命名**  
   - 建议使用小驼峰（如 `orderEntryNo`），与 URL path 保持一致，并在本模块或共通文档中公布，供其他模块拼 URL。

4. **文档维护**  
   - 目标模块新增或变更可跳转的 routeName 时，更新「routeName 与页面、参数」说明并通知调用方。

## 六、完整示例：01288 → 01206 的 orderEntryNo

- **需求**：从 01288 某页面跳转到 01206 的 orderEntryNo 页面，并传入 `corporate_code`、`table_type`。

- **源模块（01288）代码：**
  ```javascript
  var url = contextPath + '/websms01206/maintenance/orderEntryNo?corporate_code=X&table_type=order';
  window.open(url, '_blank');
  ```

- **目标模块（01206）**  
  需提供 GET：`/websms01206/maintenance/orderEntryNo`，接收 `corporate_code`、`table_type` 等 Query 参数，返回对应页面（如订单录入表单）。

## 七、模块编号与 URL 前缀一览

| 模块编号 | URL 前缀（maintenance 下） |
|----------|----------------------------|
| 01206 | {contextPath}/websms01206/maintenance/ |
| 01288 | {contextPath}/websms01288/maintenance/ |
| 012C8 | {contextPath}/websms012C8/maintenance/ |
| 01281 | {contextPath}/websms01281/maintenance/ |

跳转时在末尾加上 **routeName** 和 **?queryParams** 即可。

---

## 八、注意事项

1. **整页跳转、不用 SPA 路由**  
   转出采用 `window.open(URL, '_blank')` 打开完整 URL，不依赖前端 SPA 的路由表。

2. **参数编码**  
   所有 Query 参数值请使用 `encodeURIComponent` 或 `URLSearchParams` 编码。

3. **新窗口**  
   统一使用 `_blank`，避免覆盖当前业务页。

4. **routeName 与目标模块约定一致**  
   源模块使用的 routeName（如 `orderEntryNo`）必须与目标模块提供的 path 一致，具体以目标模块文档为准。
